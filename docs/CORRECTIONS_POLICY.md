# YETO Corrections Policy

## Purpose

This policy establishes procedures for handling corrections, disputes, and data quality issues on the YETO platform. We are committed to accuracy, transparency, and fair treatment of all parties.

## Scope

This policy applies to:
- Data accuracy corrections
- Entity matching disputes
- Content corrections
- Sanctions-related disputes
- Defamation concerns

## Principles

### 1. Accuracy First

We prioritize data accuracy above all else. When errors are identified, we correct them promptly and transparently.

### 2. Fair Process

All disputes receive fair, impartial review regardless of the submitter's identity or affiliation.

### 3. Transparency

Corrections are documented and, where appropriate, publicly noted to maintain trust.

### 4. Timeliness

We commit to responding to all submissions within defined timeframes.

## Correction Types

### Data Corrections

| Type | Description | Process |
|------|-------------|---------|
| **Factual Error** | Incorrect data value | Verify against source, correct if confirmed |
| **Outdated Data** | Data no longer current | Update with latest available |
| **Missing Data** | Data gap | Add if available from verified source |
| **Attribution Error** | Wrong source cited | Correct attribution |

### Entity Matching Corrections

| Type | Description | Process |
|------|-------------|---------|
| **False Positive** | Incorrect match | Review, mark as false positive if confirmed |
| **Missed Match** | Should have matched | Review, add match if confirmed |
| **Confidence Error** | Wrong confidence score | Recalculate and update |

### Content Corrections

| Type | Description | Process |
|------|-------------|---------|
| **Language Issue** | Non-neutral language | Editorial review and revision |
| **Translation Error** | Arabic/English mismatch | Translation review and correction |
| **Context Missing** | Insufficient context | Add appropriate context |

## Submission Process

### How to Submit

1. **Online Form**: Submit via the platform's correction request form
2. **Email**: Send to corrections@yeto.org
3. **In-Platform**: Use the "Report Issue" button on any page

### Required Information

| Field | Required | Description |
|-------|----------|-------------|
| Contact Email | Yes | For follow-up communication |
| Issue Type | Yes | Category of correction |
| Page/Data Reference | Yes | What needs correction |
| Description | Yes | Detailed explanation |
| Supporting Evidence | Recommended | Documents, links, etc. |

## Review Process

### Workflow

```
Submitted → Initial Review → Investigation → Decision → Implementation → Notification
                ↓                               ↓
          Rejected (with reason)          Additional Info Requested
```

### Review Criteria

1. **Validity**: Is the claim supported by evidence?
2. **Accuracy**: Is the current data actually incorrect?
3. **Source Quality**: Is the proposed correction from a reliable source?
4. **Impact**: What is the scope of the correction?

### Decision Options

| Decision | Action |
|----------|--------|
| **Accepted** | Correction implemented |
| **Partially Accepted** | Some aspects corrected |
| **Rejected** | No correction, with explanation |
| **Deferred** | Pending additional information |

## Timeframes

### Response Commitments

| Issue Type | Initial Response | Resolution Target |
|------------|------------------|-------------------|
| Defamation Concern | 24 hours | 3 business days |
| Data Error | 48 hours | 5 business days |
| Entity Match Dispute | 48 hours | 10 business days |
| Content Correction | 72 hours | 7 business days |

### Escalation

If resolution targets are not met:
1. Issue escalated to senior review
2. Submitter notified of delay
3. New timeline provided

## Defamation Concerns

### Special Handling

Defamation concerns receive expedited handling:

1. **Immediate Review**: Within 24 hours
2. **Temporary Action**: Content may be temporarily hidden pending review
3. **Legal Consultation**: If needed, legal counsel consulted
4. **Resolution**: Within 3 business days

### Standards

Content is considered potentially defamatory if it:
- Makes unsubstantiated claims about individuals/entities
- Uses non-neutral, characterizing language
- Implies wrongdoing without official designation
- Goes beyond factual reporting of official sources

## Documentation

### Audit Trail

All corrections maintain:
- Original submission details
- Review notes and decisions
- Implementation records
- Communication history

### Public Record

For significant corrections:
- Correction notice added to affected content
- Changelog updated
- Stakeholders notified if appropriate

## Appeals

### Appeal Process

Submitters may appeal rejected corrections:

1. Submit appeal within 30 days of decision
2. Provide additional evidence or reasoning
3. Appeal reviewed by different reviewer
4. Final decision within 10 business days

### Appeal Criteria

Appeals considered if:
- New evidence provided
- Original review missed key information
- Procedural issues in original review
