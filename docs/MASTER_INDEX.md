## Release Quality Gate Documentation

YETO now includes comprehensive release quality gates to prevent drift between documentation and implementation:

### CI/CD Pipeline
- [.github/workflows/main.yml](../.github/workflows/main.yml) - Full CI pipeline with 17 gates
- [scripts/release-gate.mjs](../scripts/release-gate.mjs) - Release gate validation script
- [scripts/registry-lint.mjs](../scripts/registry-lint.mjs) - Source registry data validation
- [eslint.config.js](../eslint.config.js) - Code quality linting

### Quality Gates (17 Checks)
1. ✅ Source Registry Count (≥250 sources)
2. ✅ Active Sources (≥150 active)
3. ✅ Sector Codebook (≥16 sectors)
4. ✅ Tier Distribution (≤70% UNKNOWN)
5. ✅ Sector Mappings (≥50% mapped)
6. ✅ No Duplicate Source IDs
7. ✅ Required Fields Complete
8. ✅ S3 Storage Health
9. ✅ Schema v2.5 Compliance
10. ✅ No Static KPIs in UI
11. ✅ No Mock Evidence Fallbacks
12. ✅ Evidence Coverage ≥95%
13. ✅ AR/EN Parity
14. ✅ Exports with Signed URLs
15. ✅ Contradiction Mode Present
16. ✅ Coverage Map Present
17. ✅ Nightly Jobs Configured
