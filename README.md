## рҹҢҚ The Challenge

> *"In Yemen, the truth is the first casualty of conflict. Economic data is fragmented, politicized, and often fabricated. Decision-makers вҖ” from humanitarian organizations to international institutions вҖ” operate in an information vacuum."*

Yemen represents one of the world's most complex economic environments:

| Challenge | Reality |
|-----------|---------|
| **Split Monetary Authority** | Since August 2016, two Central Banks operate independently in Aden (IRG) and Sana'a (DFA), each with different exchange rates, monetary policies, and fiscal realities |
| **Data Fragmentation** | Economic statistics are scattered across 292+ international sources, each with different methodologies, update frequencies, and political biases |
| **Information Warfare** | Conflicting narratives from rival authorities make it nearly impossible to establish ground truth without triangulation |
| **Humanitarian Crisis** | 21.6 million people need assistance, but aid allocation decisions rely on incomplete or outdated data |
| **No Single Source of Truth** | No platform existed that aggregated, verified, and contextualized Yemen's economic data across all sectors |

**YETO exists to solve this problem.**

---

## рҹ’Ў What YETO Does

YETO is a **full-stack economic intelligence platform** that aggregates data from 292 registered sources, processes it through 8 specialized AI agents, verifies every claim with evidence packs, and presents it through 16 sector dashboards, 5 VIP cockpits, and a comprehensive research library вҖ” all in both English and Arabic.

**Core Capabilities:**

1. **Aggregate** вҖ” Ingest economic data from the World Bank, IMF, UNHCR, OCHA, HDX, ReliefWeb, OpenAlex, and 285+ other sources through 20+ automated connectors
2. **Verify** вҖ” Every data point is linked to an Evidence Pack with source citations, confidence ratings, and contradiction detection
3. **Analyze** вҖ” 8 specialized AI agents process sector-specific data, generate context packs, detect signals, and produce evidence-backed narratives
4. **Present** вҖ” 16 sector dashboards with real-time indicators, interactive charts, and bilingual (EN/AR) content
5. **Advise** вҖ” 5 VIP cockpits deliver role-specific intelligence to the President, Finance Minister, Central Bank Governor, Humanitarian Coordinator, and Donor Analyst
6. **Publish** вҖ” Automated publication engine generates daily snapshots, weekly digests, monthly reports, and quarterly outlooks

### Data Connectors (20+ Automated)

Each connector follows the **Connector SDK** interface (`server/services/connectorSDK.ts`) with standardized methods: `fetch`, `transform`, `validate`, `deduplicate`, `store`, `backfill`, and `healthCheck`.

#### World Bank Connector (`worldBankFetcher.ts`)

Fetches Yemen-specific indicators from the World Bank API v2:

```
GET https://api.worldbank.org/v2/country/YEM/indicator/{code}?format=json&per_page=500
```

**Indicators fetched:** GDP (NY.GDP.MKTP.CD), GDP growth (NY.GDP.MKTP.KD.ZG), GDP per capita (NY.GDP.PCAP.CD), inflation (FP.CPI.TOTL.ZG), population (SP.POP.TOTL), unemployment (SL.UEM.TOTL.ZS), trade balance (NE.EXP.GNFS.ZS, NE.IMP.GNFS.ZS), FDI (BX.KLT.DINV.WD.GD.ZS), remittances (BX.TRF.PWKR.DT.GD.ZS), and 450+ more indicators across all 16 sectors.

**Data flow:** API response вҶ’ JSON parse вҶ’ year/value extraction вҶ’ duplicate check вҶ’ `time_series` table insert вҶ’ Evidence Pack creation with source citation.

#### IMF Connector (`externalDataConnectors.ts вҶ’ fetchIMFData`)

Fetches macroeconomic projections from the IMF World Economic Outlook:

```
GET https://www.imf.org/external/datamapper/api/v1/{indicator}/YEM
```

**Indicators:** Real GDP growth (NGDP_RPCH), nominal GDP (NGDPDPC), inflation (PCPIPCH), current account (BCA_NGDPD), government debt (GGXWDG_NGDP), unemployment (LUR).

#### UNHCR Connector (`externalDataConnectors.ts вҶ’ fetchUNHCRData`)

Fetches refugee and displacement data:

```
GET https://data.unhcr.org/api/v2/population/?country=YEM&limit=1000
```

**Data:** Refugee populations, IDPs, asylum seekers, returnees вҖ” broken down by year and demographic.

#### OCHA/FTS Connector (`externalDataConnectors.ts вҶ’ fetchOCHAFundingData`)

Fetches humanitarian funding flows:

```
GET https://api.hpc.tools/v2/public/fts/flow?planId=1188&limit=1000
```

**Data:** Funding requirements, commitments, disbursements by donor and sector for the Yemen Humanitarian Response Plan.

#### HDX Connector (`hdxFetcher.ts`)

Fetches 6 dataset categories from the Humanitarian Data Exchange:

- **Population** вҖ” `fetchHDXPopulation()`: Yemen population statistics
- **Food Security** вҖ” `fetchHDXFoodSecurity()`: IPC phase classifications

**Data:** Situation reports, assessments, press releases, maps вҖ” with full text extraction and sector tagging.

### Ingestion Pipeline

The ingestion pipeline processes data through 7 stages:

1. **Fetch** вҖ” Connector retrieves raw data from external API
2. **Transform** вҖ” Normalize units, dates, and field names to YETO schema
3. **Validate** вҖ” Schema validation with Zod, range checks, continuity checks
4. **Deduplicate** вҖ” Content hash comparison to prevent duplicate entries
5. **Store** вҖ” Insert into appropriate database table with provenance logging
6. **Evidence Pack** вҖ” Create evidence record linking data point to source citation
7. **Route** вҖ” Routing engine assigns data to relevant sector pages and dashboards

### Backfill System

The **Backfill Runner** (`server/services/backfillRunner.ts`) handles historical data ingestion:

- Processes data in monthly/yearly chunks with checkpoint tracking
- Idempotent inserts prevent duplicate data on re-runs
- Checkpoint table tracks progress per source per time range
- Supports resume-from-checkpoint after interruptions
- Can be triggered manually via Admin panel or scheduled via cron

### Data Quality Assurance

Every data point passes through multiple quality gates:

| Gate | Service | Description |
|------|---------|-------------|
| **Schema Validation** | `connectorSDK.ts` | Zod schema validates field types and ranges |
| **Duplicate Detection** | `connectorSDK.ts` | Content hash prevents duplicate entries |
| **Continuity Check** | `integrityCompiler.ts` | Detects gaps in time series |
| **Contradiction Detection** | `dataContradictions` table | Flags conflicting data from different sources |
| **Confidence Rating** | `confidenceRatings` table | A/B/C/D rating per data point |
| **Accuracy Checker** | `accuracyChecker.ts` | Cross-validates against known benchmarks |
| **Reliability Lab** | `reliabilityLab.ts` | Statistical tests for data reliability |
| **Registry Linter** | `registryLinter.ts` | Validates source registry completeness |

- `storeAlerts()` вҖ” Persist detected signals to database
- `notifyCriticalAlerts()` вҖ” Trigger notifications for critical signals
- `runSignalDetection()` вҖ” Full detection cycle

#### 7. Context Pack Generator (`contextPackService.ts`)

**Specialization:** Building comprehensive context packs for AI queries

**Capabilities:**
- `generateContextPack()` вҖ” Build context pack for a specific sector/topic
- `generateAllContextPacks()` вҖ” Nightly generation of all sector context packs
- `getLatestContextPack()` вҖ” Retrieve most recent context pack

#### 8. RAG Retrieval Agent (`ragService.ts`)

**Specialization:** Retrieval-Augmented Generation for evidence-backed responses

**Capabilities:**
- `retrieveContext()` вҖ” Hybrid search (keyword + embedding) across all data
- `formatContextForPrompt()` вҖ” Structure retrieved context for LLM consumption
- `extractCitations()` вҖ” Extract and validate source citations from LLM output

### Evidence Tribunal (`evidenceTribunal.ts`)

The Evidence Tribunal is the **gatekeeper** that prevents any unverified claim from reaching users:

- `collectEvidence()` вҖ” Gather all evidence for a claim from database
- `runTribunal()` вҖ” Multi-agent vote on whether evidence supports the claim
- `quickVerify()` вҖ” Fast-path verification for well-sourced claims
- **Rule:** No single agent can publish вҖ” requires tribunal consensus

### Teamwork Engine (`teamworkEngineService.ts`)

Orchestrates multi-agent collaboration:

- `createTask()` вҖ” Define a task with required agent capabilities
- `executeNextStage()` вҖ” Advance task through planner вҶ’ specialist вҶ’ verifier stages
- `submitToTribunal()` вҖ” Submit completed work for tribunal review
- `executeFullTask()` вҖ” End-to-end task execution with all stages

## рҹ“Ҡ Exchange Rate Triangulation

One of YETO's unique features is **exchange rate triangulation** for the Yemeni Rial:

```
exchangeRateService.ts вҶ’ triangulateExchangeRate()
```

Since Yemen has two competing exchange rates (Aden IRG vs Sana'a DFA), YETO:

1. Collects rates from multiple sources (CBY, parallel market, money changers)
2. Applies confidence weighting based on source reliability
3. Triangulates a "best estimate" rate with confidence interval
4. Tracks the spread between official and parallel rates
5. Detects anomalies and sudden movements
