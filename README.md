## 🌍 The Challenge

> *"In Yemen, the truth is the first casualty of conflict. Economic data is fragmented, politicized, and often fabricated. Decision-makers — from humanitarian organizations to international institutions — operate in an information vacuum."*

Yemen represents one of the world's most complex economic environments:

| Challenge | Reality |
|-----------|---------|
| **Split Monetary Authority** | Since August 2016, two Central Banks operate independently in Aden (IRG) and Sana'a (DFA), each with different exchange rates, monetary policies, and fiscal realities |
| **Data Fragmentation** | Economic statistics are scattered across 292+ international sources, each with different methodologies, update frequencies, and political biases |
| **Information Warfare** | Conflicting narratives from rival authorities make it nearly impossible to establish ground truth without triangulation |
| **Humanitarian Crisis** | 21.6 million people need assistance, but aid allocation decisions rely on incomplete or outdated data |
| **No Single Source of Truth** | No platform existed that aggregated, verified, and contextualized Yemen's economic data across all sectors |

**YETO exists to solve this problem.**

---

## 💡 What YETO Does

YETO is a **full-stack economic intelligence platform** that aggregates data from 292 registered sources, processes it through 8 specialized AI agents, verifies every claim with evidence packs, and presents it through 16 sector dashboards, 5 VIP cockpits, and a comprehensive research library — all in both English and Arabic.

**Core Capabilities:**

1. **Aggregate** — Ingest economic data from the World Bank, IMF, UNHCR, OCHA, HDX, ReliefWeb, OpenAlex, and 285+ other sources through 20+ automated connectors
2. **Verify** — Every data point is linked to an Evidence Pack with source citations, confidence ratings, and contradiction detection
3. **Analyze** — 8 specialized AI agents process sector-specific data, generate context packs, detect signals, and produce evidence-backed narratives
4. **Present** — 16 sector dashboards with real-time indicators, interactive charts, and bilingual (EN/AR) content
5. **Advise** — 5 VIP cockpits deliver role-specific intelligence to the President, Finance Minister, Central Bank Governor, Humanitarian Coordinator, and Donor Analyst
6. **Publish** — Automated publication engine generates daily snapshots, weekly digests, monthly reports, and quarterly outlooks

### Source Classification Methodology

All 292 sources have been classified into tiers using a **rule-based classification engine** with domain-specific knowledge:

**Classification Rules:**

1. **T0 — Flagship (20 sources):** International organizations with gold-standard methodology frameworks (IMF, World Bank, WTO, UN DESA, ILO, FAO, WHO, UNESCO, UNCTAD, UNIDO). These sources follow internationally recognized statistical standards (DQAF, SDDS, SNA 2008).

2. **T1 — Core (141 sources):** Primary data providers including UN agencies (UNHCR, UNICEF, OCHA, WFP, UNDP), bilateral donors (USAID, DFID/FCDO, GIZ, JICA), development banks (AfDB, IsDB, AIIB), and Yemeni government institutions (Central Bank, Ministry of Finance, Central Statistical Organization).

3. **T2 — Supplementary (103 sources):** Specialized research institutions (Brookings, Chatham House, SIPRI, V-Dem), regional organizations (GCC, Arab Monetary Fund), sector-specific bodies (OPEC, IATA), and Yemeni civil society organizations.

4. **T3 — Contextual (28 sources):** Commercial data providers (Oxford Economics, EIU, Bloomberg), local media outlets, social media monitoring, and unstructured data sources.

**Classification Confidence:** Each classification carries a confidence score (0.70–0.95) based on the specificity of the matching rule. Sources matched by exact organizational name receive higher confidence than keyword-based matches.

**Sector Assignment:** Each source is mapped to one or more of the 16 economic sectors based on its primary data domain. Cross-cutting sources (e.g., World Bank WDI) are assigned to multiple sectors.

### Data Connectors (20+ Automated)

Each connector follows the **Connector SDK** interface (`server/services/connectorSDK.ts`) with standardized methods: `fetch`, `transform`, `validate`, `deduplicate`, `store`, `backfill`, and `healthCheck`.

#### World Bank Connector (`worldBankFetcher.ts`)

Fetches Yemen-specific indicators from the World Bank API v2:

```
GET https://api.worldbank.org/v2/country/YEM/indicator/{code}?format=json&per_page=500
```

**Indicators fetched:** GDP (NY.GDP.MKTP.CD), GDP growth (NY.GDP.MKTP.KD.ZG), GDP per capita (NY.GDP.PCAP.CD), inflation (FP.CPI.TOTL.ZG), population (SP.POP.TOTL), unemployment (SL.UEM.TOTL.ZS), trade balance (NE.EXP.GNFS.ZS, NE.IMP.GNFS.ZS), FDI (BX.KLT.DINV.WD.GD.ZS), remittances (BX.TRF.PWKR.DT.GD.ZS), and 450+ more indicators across all 16 sectors.

**Data flow:** API response → JSON parse → year/value extraction → duplicate check → `time_series` table insert → Evidence Pack creation with source citation.

#### IMF Connector (`externalDataConnectors.ts → fetchIMFData`)

Fetches macroeconomic projections from the IMF World Economic Outlook:

```
GET https://www.imf.org/external/datamapper/api/v1/{indicator}/YEM
```

**Indicators:** Real GDP growth (NGDP_RPCH), nominal GDP (NGDPDPC), inflation (PCPIPCH), current account (BCA_NGDPD), government debt (GGXWDG_NGDP), unemployment (LUR).

#### UNHCR Connector (`externalDataConnectors.ts → fetchUNHCRData`)

Fetches refugee and displacement data:

```
GET https://data.unhcr.org/api/v2/population/?country=YEM&limit=1000
```

**Data:** Refugee populations, IDPs, asylum seekers, returnees — broken down by year and demographic.

#### OCHA/FTS Connector (`externalDataConnectors.ts → fetchOCHAFundingData`)

Fetches humanitarian funding flows:

```
GET https://api.hpc.tools/v2/public/fts/flow?planId=1188&limit=1000
```

**Data:** Funding requirements, commitments, disbursements by donor and sector for the Yemen Humanitarian Response Plan.

#### HDX Connector (`hdxFetcher.ts`)

Fetches 6 dataset categories from the Humanitarian Data Exchange:

- **Population** — `fetchHDXPopulation()`: Yemen population statistics
- **Food Security** — `fetchHDXFoodSecurity()`: IPC phase classifications
- **IDPs** — `fetchHDXIDPs()`: Internal displacement tracking
- **Refugees** — `fetchHDXRefugees()`: Cross-border displacement
- **Funding** — `fetchHDXFunding()`: Humanitarian funding flows
- **Operational Presence** — `fetchHDXOperationalPresence()`: Who does what where (3W)

#### ReliefWeb Connector (`reliefwebFetcher.ts`)

Fetches humanitarian reports and situation updates:

```
GET https://api.reliefweb.int/v1/reports?filter[field]=country.name&filter[value]=Yemen
```

**Data:** Situation reports, assessments, press releases, maps — with full text extraction and sector tagging.

#### OpenAlex Connector (Literature Backfill)

Fetches academic publications about Yemen's economy:

```
GET https://api.openalex.org/works?filter=default.search:yemen+economy&per_page=200
```

**Data:** 372 Yemen-specific academic papers, working papers, and reports spanning 2010–2026, with abstracts, authors, publishers, and DOI links.

### Ingestion Pipeline

The ingestion pipeline processes data through 7 stages:

1. **Fetch** — Connector retrieves raw data from external API
2. **Transform** — Normalize units, dates, and field names to YETO schema
3. **Validate** — Schema validation with Zod, range checks, continuity checks
4. **Deduplicate** — Content hash comparison to prevent duplicate entries
5. **Store** — Insert into appropriate database table with provenance logging
6. **Evidence Pack** — Create evidence record linking data point to source citation
7. **Route** — Routing engine assigns data to relevant sector pages and dashboards

## 🤖 AI Agent Society

YETO operates a **multi-agent system** where 8 specialized AI agents collaborate through a teamwork engine, with an Evidence Tribunal ensuring no unverified claims reach users.

### The 8 Agents

#### 1. Macroeconomic Agent (`macroSectorAgent.ts`)

**Specialization:** GDP, inflation, exchange rates, monetary policy, fiscal balance

**Capabilities:**
- `buildMacroContextPack()` — Assembles current macro indicators with historical context
- `generateDailyDigest()` — Produces daily macro summary with key movements
- `generateWeeklyBrief()` — Weekly macroeconomic analysis with trend identification
- `triageMacroAlerts()` — Prioritizes macro signals by severity and impact
- `runMacroAgent()` — Full agent execution cycle

**Data Sources:** World Bank WDI, IMF WEO, CBY directives, exchange rate feeds

#### 2. Labor & Wages Agent (`laborWagesAgent.ts`)

**Specialization:** Employment, wages, labor force participation, livelihoods

**Capabilities:**
- `buildLaborContextPack()` — Labor market indicators with regime-split analysis
- `generateDailyLivelihoodsDigest()` — Daily livelihoods situation update
- `generateWeeklyBulletin()` — Weekly labor market bulletin
- `generateMonthlyMonitor()` — Monthly employment and wages monitor
- `generateWageAdequacyShockNote()` — Alert when wages fall below adequacy thresholds

**Data Sources:** ILO, World Bank labor indicators, WFP food basket prices

#### 3. Poverty & Human Development Agent (`povertyHumandevAgent.ts`)

**Specialization:** Poverty rates, HDI, education, health, food security

**Capabilities:** Context pack generation, poverty trend analysis, HDI decomposition, food security alerts

**Data Sources:** World Bank PIP, UNDP HDI, UNICEF, WHO, WFP

#### 4. Sector Agent Service (`sectorAgentService.ts`)

**Specialization:** Cross-sector analysis and sector-specific intelligence

**Capabilities:**
- `getSectorDefinition()` — Retrieve sector metadata and mission
- `getSectorIndicators()` — Fetch all indicators for a sector
- `getSectorContradictions()` — Identify conflicting data within a sector
- `getSectorGaps()` — Identify missing data coverage
- `calculateSectorCoverage()` — Compute data completeness score
- `generateSectorContextPack()` — Build comprehensive sector context

#### 5. Banking Knowledge Agent (`bankingKnowledgeBase.ts`)

**Specialization:** Banking sector, commercial banks, CBY directives, financial inclusion

**Data:** 39 commercial bank profiles, CBY directive tracking, banking sector metrics

#### 6. Signal Detector (`signalDetector.ts`)

**Specialization:** Anomaly detection, trend breaks, threshold breaches

**Capabilities:**
- `detectSignals()` — Scan all indicators for anomalies
- `storeAlerts()` — Persist detected signals to database
- `notifyCriticalAlerts()` — Trigger notifications for critical signals
- `runSignalDetection()` — Full detection cycle

#### 7. Context Pack Generator (`contextPackService.ts`)

**Specialization:** Building comprehensive context packs for AI queries

**Capabilities:**
- `generateContextPack()` — Build context pack for a specific sector/topic
- `generateAllContextPacks()` — Nightly generation of all sector context packs
- `getLatestContextPack()` — Retrieve most recent context pack

#### 8. RAG Retrieval Agent (`ragService.ts`)

**Specialization:** Retrieval-Augmented Generation for evidence-backed responses

**Capabilities:**
- `retrieveContext()` — Hybrid search (keyword + embedding) across all data
- `formatContextForPrompt()` — Structure retrieved context for LLM consumption
- `extractCitations()` — Extract and validate source citations from LLM output

### Evidence Tribunal (`evidenceTribunal.ts`)

The Evidence Tribunal is the **gatekeeper** that prevents any unverified claim from reaching users:

- `collectEvidence()` — Gather all evidence for a claim from database
- `runTribunal()` — Multi-agent vote on whether evidence supports the claim
- `quickVerify()` — Fast-path verification for well-sourced claims
- **Rule:** No single agent can publish — requires tribunal consensus

### Teamwork Engine (`teamworkEngineService.ts`)

Orchestrates multi-agent collaboration:

- `createTask()` — Define a task with required agent capabilities
- `executeNextStage()` — Advance task through planner → specialist → verifier stages
- `submitToTribunal()` — Submit completed work for tribunal review
- `executeFullTask()` — End-to-end task execution with all stages

## 📊 Exchange Rate Triangulation

One of YETO's unique features is **exchange rate triangulation** for the Yemeni Rial:

```
exchangeRateService.ts → triangulateExchangeRate()
```

Since Yemen has two competing exchange rates (Aden IRG vs Sana'a DFA), YETO:

1. Collects rates from multiple sources (CBY, parallel market, money changers)
2. Applies confidence weighting based on source reliability
3. Triangulates a "best estimate" rate with confidence interval
4. Tracks the spread between official and parallel rates
5. Detects anomalies and sudden movements
